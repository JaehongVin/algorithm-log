# Algorithm Log

> 학습 플랜·맥락은 `data.md`를 참고한다. (git 미추적 파일)

알고리즘 코딩테스트 풀이 기록 저장소.

## 디렉토리 구조

```
algorithm-log/
├── 프로그래머스/
│   ├── lv0/
│   ├── lv1/
│   ├── lv2/
│   ├── lv3/
│   └── lv4/
└── (추후 플랫폼 추가 예정)
```

## 규칙

- 플랫폼 단위로 최상위 디렉토리 구분
- 플랫폼 하위에 레벨별 디렉토리 (`lv0`, `lv1`, `lv2`, `lv3`, `lv4`)
- 파일명은 **한글 문제명**으로 `.js` 확장자 사용
  - 예: `프로그래머스/lv1/두 수의 합.js`
- 현재 지원 플랫폼: 프로그래머스
- 추후 플랫폼 추가 시 동일한 레벨 구조 유지

## 파일 형식

특별히 정해진 템플릿 없음. 풀이 코드를 그대로 저장.

## 코드 평가 방식

코드 평가 요청 시 아래 4가지 기준으로 냉정하게 평가한다.

1. **자료구조** — 선택한 자료구조가 문제에 적합한지, 더 나은 대안이 있는지
2. **알고리즘** — 접근법이 이 문제 유형에 맞는지 (탐색, DP, 그리디, 정렬 등)
3. **시간복잡도** — 현재 Big-O를 명시하고 최적 복잡도 대비 효율적인지 판단
4. **보편성** — 코딩테스트에서 일반적으로 통용되는 풀이인지 아닌지

평가 후 보편적인 정답 풀이를 코드로 제시한다.
